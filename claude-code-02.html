Claude Code 命令行界面的完整参考，包括命令和标志。

CLI 命令

命令 描述 示例
claude 启动交互式 REPL claude
claude "query" 使用初始提示启动 REPL claude "explain this project"
claude -p "query" 通过 SDK 查询，然后退出 claude -p "explain this function"
cat file | claude -p "query" 处理管道内容 cat logs.txt | claude -p "explain"
claude -c 继续最近的对话 claude -c
claude -c -p "query" 通过 SDK 继续 claude -c -p "Check for type errors"
claude -r "<session-id>" "query" 通过 ID 恢复会话 claude -r "abc123" "Finish this PR"
claude update 更新到最新版本 claude update
claude mcp 配置模型上下文协议 (MCP) 服务器 claude mcp

CLI 标志

标志 描述 示例
--add-dir 添加额外的工作目录供 Claude 访问 claude --add-dir ../apps ../lib
--allowedTools 应该在不提示用户许可的情况下允许的工具列表 "Bash(git log:*)" "Read"
--disallowedTools 应该在不提示用户许可的情况下禁止的工具列表 "Bash(git diff:*)" "Edit"
--print, -p 打印响应而不使用交互模式 claude -p "query"
--output-format 指定打印模式的输出格式（text、json、stream-json） claude -p "query" --output-format json
--verbose 启用详细日志记录 claude --verbose
--model 设置使用的模型 claude --model claude-sonnet-4-20250514
--permission-mode 在指定的权限模式下开始 claude --permission-mode plan
--continue 在当前目录中加载最近的对话 claude --continue
--dangerously-skip-permissions 跳过权限提示（谨慎使用） claude --dangerously-skip-permissions

⚡ 斜杠命令：高效操作的快捷键

斜杠命令是Claude Code中的"魔法咒语"，让你能够快速执行特定操作，极大提升工作效率。

命令 分类 功能描述 使用场景
/clear 对话管理 清空当前对话历史，释放上下文空间 开始新任务时
/compact 对话管理 智能压缩对话历史，保留关键信息 长会话优化
/init 项目管理 分析代码库并生成CLAUDE.md文件 项目初始化
/cost 成本管理 查看当前会话Token使用量和费用 成本监控
/model 配置管理 切换使用的Claude模型 性能调优
/mcp 扩展功能 管理模型上下文协议服务器 能力扩展

⚙️ Settings设置：五层配置体系

🎯 什么是Settings设置？

Settings设置是Claude Code的核心配置系统，它决定了AI代理如何与您的代码库交互。通过精细的配置管理，您可以：

  • 控制AI行为：设置AI使用的模型、温度参数、最大token数等
  • 管理工具权限：精确控制AI可以执行哪些操作，保障代码安全
  • 定制工作流：适配团队的编码规范和项目特定需求
  • 平衡效率与安全：在自动化便利性和操作安全性之间找到最佳平衡点

🧠 内存管理：AI的项目记忆系统

📝 什么是CLAUDE.md文件？

CLAUDE.md文件是Claude Code的"大脑"，它是一个特殊的Markdown文件，相当于给AI的项目说明书和行为指南。当Claude Code启动时，会自动读取这个文件，将其内容作为高优先级的上下文信息。

🎯 CLAUDE.md的作用

📖

项目文档

记录项目技术栈、架构决策、核心模块，让AI快速理解代码库

📏

编码规范

定义代码风格、命名约定、测试要求，确保AI生成的代码符合团队标准

⚡

常用命令

记录构建、测试、部署等常用命令，让AI能够直接执行项目操作

🔄

工作流程

说明Git分支策略、提交规范、发布流程，让AI遵循团队协作模式

💡 为什么需要CLAUDE.md？

解决AI"健忘"问题

没有CLAUDE.md时，每次与AI交互都需要重新解释项目背景、技术栈、编码规范等信息，不仅浪费时间，还消耗大量token。 CLAUDE.md文件实现了"一次告知，永久遵循"的效果，让AI成为真正了解你项目的智能助手。

CLAUDE.md文件系统是Claude Code最具革命性的功能之一，它为AI提供持久化的项目记忆，让每次交互都基于完整的上下文。

⚠️ 重要更新

./CLAUDE.local.md 已被废弃：根据官方文档，本地项目内存文件 ./CLAUDE.local.md 已不再支持。 现在推荐使用项目级和用户级的CLAUDE.md文件来管理项目记忆。

📁 双层记忆架构

🌐 MCP协议：连接万物的AI接口

模型上下文协议（MCP）是Claude Code的"终极武器"，它为AI打开了通往整个数字世界的大门。

Claude Code

通过MCP协议连接各种外部服务

🛠️ MCP 命令列表

命令 功能描述 使用示例
claude mcp 启动MCP配置向导 claude mcp
claude mcp add 添加新的MCP服务器 claude mcp add postgres-server
claude mcp list 列出已配置的MCP服务器 claude mcp list
claude mcp remove 移除指定的MCP服务器 claude mcp remove server-name
claude mcp status 查看MCP服务器连接状态 claude mcp status
claude mcp test 测试MCP服务器连接 claude mcp test server-name
MCP配置示例
# 添加PostgreSQL MCP服务器
claude mcp add pg_server /path/to/postgres-mcp-server \
  --connection-string "postgresql://user:pass@host:port/db"

# 使用MCP查询数据库
> 请描述 @pg_server:schema://users 表的结构

# 查询业务数据
> 使用 @pg_server:db://sales_db 查询上个季度销售额最高的三个产品

# GitHub集成示例
> 分析 @github:issue://anthropics/claude-code/123 这个问题
> 执行 /mcp__github__review_pr 456 审查这个PR
