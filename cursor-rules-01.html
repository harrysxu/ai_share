🎯 什么是Cursor Rules

💡 核心概念

Cursor Rules是一种为AI编程助手提供持久化、可复用上下文的精密机制。它不仅仅是简单的指令集，而是将AI从通用代码生成器转变为深度理解项目特定领域知识、架构决策和编码风格的专业伙伴。

在人工智能辅助开发的浪潮中，Cursor IDE以其深度集成AI的能力脱颖而出。其核心特性之一——"Rules"（规则），超越了聊天框中一次性的提示，旨在为AI编程伙伴提供系统级指导。

🎯 持久化上下文

规则提供持久化、可复用的上下文，解决大型语言模型固有的"无状态"问题，让AI在不同会话间保持一致的理解。

🧠 智能化指导

通过规则，AI能够理解项目的特定架构、编码风格和业务逻辑，提供更精准的代码建议和生成。

⚡ 效率提升

避免重复解释项目规范，让开发者专注于核心业务逻辑，显著提升开发效率和代码质量。

🤝 团队协作

规则可以版本控制和团队共享，确保所有成员和AI助手都遵循统一的开发标准。

🔧 Cursor Rules解决了什么问题

⚡ 核心价值：一致性、高质量、自动化

通过规则，开发者可以将关于代码库的领域特定知识、项目特有的工作流程或模板、以及标准化的代码风格或架构决策进行编码，最终带来更高的代码质量、更强的团队协作一致性，并显著提升开发效率。

🎯 主要解决的问题

❌ AI"健忘症"

问题：大型语言模型在会话间不保留记忆
解决：规则提供持久化上下文，确保AI始终记住项目规范

❌ 重复解释

问题：每次对话都需要重新解释项目背景
解决：规则自动注入相关上下文，减少重复性工作

❌ 代码不一致

问题：AI生成的代码风格和架构不统一
解决：通过规则强制执行编码标准和架构模式

❌ 团队协作困难

问题：团队成员使用AI的方式不统一
解决：共享规则确保团队使用统一的AI指导原则

✅ 令牌经济学优化

规则中的每一个词都会消耗大型语言模型的令牌，这会减少可用于理解代码和用户查询的上下文窗口容量。因此，最高效的规则编写者，并非提供信息最多的人，而是提供最相关、最简洁信息的人。

📋 Cursor Rules的类型与工作原理

🏗️ 三层规则体系

Cursor提供了多种作用域的规则，形成一个层次化的上下文系统，以适应从个人偏好到团队规范的各种需求。

🌐 规则作用域分类

👤 User Rules (用户规则)

作用域：全局，所有项目
存储：Cursor设置中
用途：个人编码风格、响应语调、通用编程习惯

// 示例：全局用户规则
- 注释使用中文
- 优先使用函数式编程
- 变量命名使用驼峰式

📁 Project Rules (项目规则)

作用域：特定项目
存储：.cursor/rules文件夹
用途：项目特定的技术栈、架构模式、团队规范

// 示例：项目规则
- 使用TypeScript
- 遵循Next.js最佳实践
- 使用Tailwind CSS进行样式设计

🧠 Memories (记忆)

作用域：对话级别
存储：自动生成
用途：AI根据对话自动学习和归纳的决策和上下文

📄 .cursorrules (遗留)

作用域：项目根目录
存储：单一文件
状态：已弃用，建议迁移到新格式

⚙️ 规则激活机制

两阶段激活过程
阶段一：注入
→
阶段二：激活

规则首先被注入到系统提示中，然后AI根据相关性决定是否激活使用

📄 .mdc文件格式详解

🎯 MDC格式概述

MDC (Markdown Configuration) 是Cursor专用的规则文件格式，结合了YAML前置元数据和Markdown内容，提供了强大而灵活的规则定义能力。

🏗️ 文件结构解析

.mdc文件结构
---
description: 规则描述（AI用于判断是否应用此规则）
globs: ["文件匹配模式"]
alwaysApply: true/false
---

# 规则标题

## 规则内容
- 使用Markdown语法编写
- 可以包含代码示例
- 支持@文件引用

@template-file.ts

📋 前置元数据字段详解

字段名 类型 必需 说明
description 字符串 推荐 AI用于判断规则相关性的描述
globs 数组 可选 文件匹配模式，支持通配符
alwaysApply 布尔值 可选 是否始终应用此规则

🎨 Markdown内容编写最佳实践

📝 结构化组织

使用清晰的标题层次结构，利用Markdown的格式化功能组织内容，包括标题、列表、代码块等。

💡 具体示例

提供具体的代码示例，特别是"正反对比"的示例，明确展示期望和禁止的代码模式。

🔗 文件引用

使用@符号引用项目中的特定文件，为AI提供具体的上下文锚点。

⚡ 简洁明了

保持规则简洁，目标是每个规则文件少于500行，避免信息过载。

🛠️ 创建和管理规则

🎮 UI驱动创建

使用命令面板（Cmd+Shift+P）执行"New Cursor Rule"命令，或在"Cursor Settings > Rules"界面操作。

💬 聊天驱动生成

使用/Generate Cursor Rules命令，让AI根据当前对话内容直接生成规则。

✏️ 手动编辑

直接创建.mdc文件，完全控制文件结构和内容，适合复杂规则。

🔧 批量管理

使用社区工具如vscode-cursor-rules扩展，从awesome-cursorrules仓库批量获取规则。

# TypeScript React 开发规范

## 组件开发规则


### ❌ 避免的做法
```typescript

  return ;

```

## 样式规范
- 使用Tailwind CSS进行样式设计
- 避免内联样式
- 遵循移动优先的响应式设计

## 文件引用
@src/components/ui/Button.tsx
@docs/component-guidelines.md

## 测试要求
- 每个组件必须有对应的测试文件
- 测试覆盖率不低于80%
- 使用Jest和React Testing Library

⚠️ 常见错误避免

  • 前置元数据必须位于文件顶部，使用正确的YAML格式
  • JSON格式的注解必须是有效的JSON
  • 避免引用不存在的文件，确保所有@引用的文件都存在
  • 保持规则简洁，避免信息过载影响AI理解
