⚙️ Settings设置

五层配置体系，精确控制AI行为

🎯 什么是Settings设置？
Settings设置是Claude Code的核心配置系统，通过分层设置文件（settings.json）来控制AI代理如何与您的代码库交互。 它决定了权限管理、工具访问、环境变量等关键行为，在自动化便利性和操作安全性之间找到最佳平衡点。

Settings的核心作用

🛡️

安全控制

通过精细的权限规则控制AI可以执行的操作。

⚡

性能优化

配置模型选择、清理周期、输出限制来优化响应质量。

👥

团队协作

项目级配置确保团队成员使用一致的AI行为标准。

五层配置文件体系

1. 企业策略
系统级强制设置
最高优先级
2. 命令行参数
临时覆盖设置
单次有效
3. 本地项目
.claude/settings.local.json
个人项目配置
4. 共享项目
.claude/settings.json
团队共享
5. 用户全局
~/.claude/settings.json
最低优先级

可用设置选项

设置项 描述 示例值 操作
permissions 权限控制配置 {"allow": [...], "deny": [...]}
env 环境变量配置 {"NODE_ENV": "production"}
model 默认使用的模型 "claude-3-5-sonnet-20241022" -
cleanupPeriodDays 聊天记录保留天数 30 -
hooks 工具执行前后的钩子 {"PreToolUse": {...}} -
forceLoginMethod 强制登录方式 "claudeai" 或 "console" -
enableAllProjectMcpServers 自动批准所有项目MCP服务器 true / false -
完整settings.json示例
{
  "permissions": {
    "allow": [
      "Bash(npm run lint)",
      "Bash(npm run test:*)",
      "Read(~/.zshrc)",
      "Edit(src/**/*.ts)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(curl:*)"
    ],
    "additionalDirectories": [
      "../docs/",
      "../shared/"
    ],
    "defaultMode": "acceptEdits"
  },
  "env": {
    "NODE_ENV": "development",
    "CLAUDE_CODE_ENABLE_TELEMETRY": "1"
  },
  "model": "claude-3-5-sonnet-20241022",
  "cleanupPeriodDays": 30,
  "hooks": {
    "PreToolUse": {
      "Bash": "echo 'Running command...'"
    }
  }
}

🧠 内存管理

AI的项目记忆系统

📝 什么是CLAUDE.md文件？
CLAUDE.md文件是Claude Code的"大脑"，它是一个特殊的Markdown文件， 相当于给AI的项目说明书和行为指南。当Claude Code启动时， 会自动读取这个文件，将其内容作为高优先级的上下文信息。

内存文件类型与位置

文件类型 路径 作用域 Git状态
项目内存 ./CLAUDE.md 团队共享的项目宪法 ✅ 提交到Git
用户内存 ~/.claude/CLAUDE.md 个人全局偏好设置 ❌ 不提交
本地项目内存 ./CLAUDE.local.md 个人项目临时笔记 ⚠️ 已废弃
⚠️ 重要更新
./CLAUDE.local.md 已被废弃：根据官方文档， 本地项目内存文件 ./CLAUDE.local.md 已不再支持。 现在推荐使用项目级和用户级的CLAUDE.md文件来管理项目记忆。

🌐 MCP协议

模型上下文协议 - 连接万物的AI接口

🎯 什么是MCP协议？
Model Context Protocol (MCP) 是由Anthropic开发的开放标准，用于标准化AI应用程序如何向大型语言模型提供上下文。 它就像AI的"USB-C接口"，让模型能够轻松连接到各种数据源和工具，如文件系统、数据库和API。

MCP架构原理

客户端-主机-服务器架构

基于JSON-RPC 2.0的状态会话协议

🖥️

主机 (Host)

作为容器和协调器：
• 创建和管理多个客户端实例
• 控制连接权限和生命周期
• 执行安全策略和用户授权
• 协调AI/LLM集成和采样

🔗

客户端 (Client)

维护独立的服务器连接：
• 与特定服务器建立1对1会话
• 处理协议协商和能力交换
• 双向路由协议消息
• 管理订阅和通知

⚙️

服务器 (Server)

提供专门的上下文和功能：
• 通过MCP原语暴露资源、工具和提示
• 独立运行，专注特定职责
• 通过客户端接口请求采样
• 遵守安全约束

MCP三大核心功能

📝

Prompts (提示)

预定义的模板或指令：
• 用户控制的交互模式
• 可通过斜杠命令触发
• 支持参数化定制
• 结构化消息格式

📊

Resources (资源)

结构化数据或内容：
• 应用程序驱动的上下文
• 通过URI唯一标识
• 支持订阅变更通知
• 文件、数据库模式等

🛠️

Tools (工具)

可执行的函数功能：
• 模型控制的动态调用
• 与外部系统交互
• 查询数据库、调用API
• 执行计算和操作

MCP配置示例 (.mcp.json)
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "your-token"
      }
    },
    "postgres": {
      "command": "node",
      "args": ["/path/to/postgres-mcp-server"],
      "env": {
        "DATABASE_URL": "postgresql://user:pass@host:port/db"
      }
    }
  }
}
